Task 3: Creating Power Automate flows for GitHub

In this section, you will learn how to create Power Automate flows to automate GitHub actions. Follow the steps below to gain hands-on experience and understand automation.

1 log In to power-automate

↓

Go to the power-automate , to start the practical part.

2 Create a new flow in Power Automate

↓

To create a new flow in Power Automate:

  1. Click on the "Create" button in the left-hand menu.
  2. Select "Automated flow" from the options.
  3. Provide a name for your flow and click "Create".

3 Add a trigger for GitHub actions

↓

To add a trigger for GitHub actions:

  1. In the flow designer, click on "New step".
  2. Search for "GitHub" and select the GitHub connector.
  3. Choose a trigger, such as "When an issue is created" or "When a pull request is merged".
  4. Authenticate with your GitHub account if prompted.

4 Add actions to the flow

↓

To add actions to the flow:

  1. Click on "New step" after adding the trigger.
  2. Search for the desired action, such as "Create an issue" or "Send an email".
  3. Configure the action with the necessary details.
  4. Repeat the process to add more actions as needed.

5 Provide examples of common GitHub actions

↓

Here are some examples of common GitHub actions that can be automated using Power Automate:

  • Automatically create an issue when a new pull request is opened.
  • Send a notification to a Slack channel when an issue is assigned to you.
  • Label pull requests based on the files changed.
  • Close stale issues after a certain period of inactivity.
