Task 3: Creating Power Automate flows for GitHub

In this section, you will learn how to create Power Automate flows to automate GitHub actions. Follow the steps below to gain hands-on experience and understand automation.

1 log In to power-automate

↓

Go to the power-automate , to start the practical part.

2 Create a new flow in Power Automate

↓

To create a new flow in Power Automate:

  1. Click on the "Create" button in the left-hand menu.
  2. Select "Automated flow" from the options.
  3. Provide a name for your flow and click "Create".

3 Add a trigger for GitHub actions

↓

To add a trigger for GitHub actions:

  1. In the flow designer, click on "New step".
  2. Search for "GitHub" and select the GitHub connector.
  3. Choose a trigger, such as "When an issue is created" or "When a pull request is merged".
  4. Authenticate with your GitHub account if prompted.

4 Add actions to the flow

↓

To add actions to the flow:

  1. Click on "New step" after adding the trigger.
  2. Search for the desired action, such as "Create an issue" or "Send an email".
  3. Configure the action with the necessary details.
  4. Repeat the process to add more actions as needed.

5 Provide examples of common GitHub actions

↓

Here are some examples of common GitHub actions that can be automated using Power Automate:

  • Automatically create an issue when a new pull request is opened.
  • Send a notification to a Slack channel when an issue is assigned to you.
  • Label pull requests based on the files changed.
  • Close stale issues after a certain period of inactivity.

Task 3: Creating Power Automate flows for GitHub

In this section, you will learn how to create Power Automate flows to automate GitHub actions. Follow the steps below to gain hands-on experience and understand automation.

1 Log In to Power Automate

↓

Go to Power Automate to start the practical part.

2 Connect to GitHub

↓

To automate GitHub actions, you first need to establish a connection between Power Automate and GitHub:

  1. From the Power Automate homepage, click on "Create" in the left sidebar
  2. Select "Instant cloud flow"
  3. Choose a trigger (for this workshop, select "Manually trigger a flow")
  4. Click "Create"

Now that you've created a flow, you need to add a GitHub action:

  1. Click on "New step"
  2. Search for "GitHub" in the connectors search box
  3. Select "GitHub" from the results
  4. You'll be prompted to sign in and authorize Power Automate to access your GitHub account

3 Create a PR Review Notification Flow

↓

Let's create a flow that notifies you when a new pull request is created or needs your review:

  1. From the Power Automate homepage, click on "Create" in the left sidebar
  2. Select "Automated cloud flow"
  3. Search for "GitHub" and select "When a pull request is created"
  4. Select your repository
  5. Click "New step" and search for "Send an email"
  6. Configure the email action with dynamic content from the GitHub trigger

4 Create a PR Auto-Approval Flow for Renovate

↓

Now let's create a more advanced flow that automatically approves pull requests from Renovate if they meet certain criteria:

  1. Create a new automated flow with the trigger "When a pull request is created" (GitHub)
  2. Add a "Condition" control to check if the PR creator is "renovate[bot]"
  3. In the "If yes" branch, add the GitHub action "Submit a pull request review"
  4. Configure it with:
    • Repository: [your repository]
    • Pull request number: [dynamic content from trigger]
    • Event: APPROVE
    • Body: "Automatically approved Renovate PR"

↓

6 Additional Flow Ideas

↓

Here are some additional GitHub automation flows you can create:

  • Issue Assignment: Automatically assign issues with specific labels to team members
  • PR Reminders: Send reminders for PRs that have been open without review for more than 24 hours
  • Branch Protection: Get notified when someone tries to push directly to the main branch
  • Issue Templates: Auto-label issues based on their template type
  • Release Notifications: Send team-wide notifications when a new release is published

Challenge: Try creating one of these additional flows on your own!
